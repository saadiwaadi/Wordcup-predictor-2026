[03] WHAT THESE NUMBERS MEAN

GUIDE
OUTCOME ACCURACY
Win / Draw / Loss prediction. The model picks whichever outcome has the highest probability. 60% means 6 in 10 matches called correctly. Random chance = 33%.
BRIER SCORE
Measures probability quality, not just the winner call. Penalises confident wrong predictions. Perfect = 0.000, Random = 0.667. Our 0.551 beats random — probabilities have real signal.
CALIBRATION
When the model says 60% confidence, does it win ~60% of those matches? IMPROVING means higher confidence = higher actual accuracy. This is the strongest sign the model is doing real work.
VS RANDOM / VS NAIVE
Random = predict HOME/DRAW/AWAY randomly (~33%). Naive = always pick better FIFA rank. Our model must beat both to prove it adds value beyond simple heuristics.
WHY DRAW ACCURACY = 0%
Poisson models structurally under-predict draws. Draw probability (~30%) rarely exceeds win probability (~40-60%), so the model never picks draws as its top outcome. This is a known mathematical limitation — not a bug.

ORACLE-26 // SYSTEM ARCHITECTURE
WHAT IS ORACLE-26?
ORACLE-26 is a mathematical match prediction system for the 2026 FIFA World Cup. It uses three layers of data to generate win probabilities and expected scorelines for any matchup between the 48 competing nations.
LAYER 1 — STATIC STRENGTH
Pre-tournament baseline derived from three signals weighted and combined:
Elo rating × 1.00 weight FIFA rank × 0.50 weight Market value × 0.25 weight ───────────────────────────── P_static = weighted average → [0.05, 0.95] Host nation bonus: +8% additive
LAYER 2 — DYNAMIC FORM
Adjusts static strength using live tournament data and recent results:
Form ratio = points from last 6 / 18 Elo momentum = elo_delta_90 / 100 Injury penalty = -5% if key player out P_dynamic = P_static × 0.65 + form_signal × 0.35 + momentum × 0.02 - injury_penalty
LAYER 3 — POISSON GOAL MODEL
Converts win probability into expected goals (lambda) and generates a 7x7 scoreline probability matrix:
IF team has 2+ matches played: λ = avgGoalsFor × 0.6 + (1.8 × P_dynamic + 0.27) × 0.4 IF no live data yet: λ = 1.8 × P_dynamic + 0.27 Residual correction: Home: λ += +0.162 (from backtest) Away: λ += -0.081 (from backtest) Dixon-Coles tau correction applied to 0-0, 1-0, 0-1, 1-1 scorelines. RHO = -0.04
DRAW PROBABILITY
Computed via Gaussian distribution centred at 0.5 (even matchups):
P_draw = (1/3) × e^( -(P_dynamic - 0.5)² / (2 × 0.28²) ) Note: Poisson models structurally under-predict draws. Draw accuracy is a known limitation of this approach.
CONFIDENCE SCORE
Trust score from 0-100 based on data quality penalties:
Start: 100 Tier 4 team: -20 each Under 3 WC apps: -15 each No H2H record: -10 Stale data: -15 Key injury: -5 each Neutral venue: -5 always ───────────────────────────── 85+ = HIGH 65+ = MEDIUM 45+ = LOW 0+ = VERY LOW

MATHEMATICAL LIMITATIONS
Known weaknesses in current model:
1. Lambda circularity — probability feeds lambda which generates probability. Attack/defence parameters planned for v2. 2. Draw under-prediction — Poisson structurally misses draws. 0% draw accuracy is expected. 3. Small sample — 60 matches is statistically limited. Calibration improves as tournament progresses. 4. No live player data — xG values are pre-tournament estimates for most teams except FRA and ARG.
